## 1. What and Why of DevOps?

### What is DevOps?
DevOps is a set of practices that combines software development (Dev) and IT operations (Ops). It aims to shorten the systems development life cycle and provide continuous delivery with high software quality.

### Why DevOps?
- **Speed**: Move at high velocity to innovate for customers faster.
- **Rapid Delivery**: Increase the frequency and pace of releases.
- **Reliability**: Ensure the quality of application updates and infrastructure changes.
- **Scale**: Manage infrastructure and development processes at scale with automation.
- **Improved Collaboration**: Build more effective teams with shared ownership.

**Jenkinsfile:**

```groovy
    agent any
    stages {
        stage('Stage 1: Git Clone') { steps { checkout scm } }
        stage('Stage 2: Maven Build') { steps { sh 'mvn clean package' } }
        stage('Stage 4: Build Docker Image') { steps { sh 'docker build -t abhashti/calculator:latest .' } }
        stage('Stage 5: Push Docker Image') { 
            steps { 
                withCredentials([usernamePassword(credentialsId: 'dockerhub-creds', ...)]) { 
                    sh 'docker push abhashti/calculator:latest' 
                } 
            } 
        }
    }
}
```

### Step 3.4: Containerization (Docker)


**Commands:**
```bash

```

---

### Step 3.5: Deployment (Ansible)
**Brief:** Ansible automates the container deployment on the remote server.

**Playbook (deploy.yml):**
```yaml
- hosts: localhost
  tasks:
    - name: Pull image
      shell: docker pull abhashti/calculator:latest
    - name: Running container
      shell: docker run -it -d --name Calculator -p 8081:8080 abhashti/calculator:latest
```

## 4. User Interface for "Noobs"

- **Operation Selection**: Buttons for Square Root, Factorial, Natural Log, and Power.
- **Dynamic Inputs**: Input fields automatically adjust based on the selected operation.
- **Instant Results**: Results are displayed directly on the page without reloading.
- **Exit Option**: A dedicated exit button to close the session.

## 5. How to Use (Simplified Guide)

### Option A: Web Browser (Simplest)
1. **Pull the Image**: `docker pull abhashti/calculator:latest`
2. **Run the Container**: `docker run -d --name my-calc -p 8081:8080 abhashti/calculator:latest`
3. **Open Browser**: Navigate to `http://localhost:8081` in your favorite web browser.
4. **Calculate**: Enter your numbers and click the operations!

### Option B: Command Line (For Techies)
Use `curl` to interact with the API endpoints directly:
- **Square Root**: `curl "http://localhost:8081/calculator/sqrt?x=16"`
- **Factorial**: `curl "http://localhost:8081/calculator/factorial?x=5"`
- **Power**: `curl "http://localhost:8081/calculator/power?base=2&exp=3"`
