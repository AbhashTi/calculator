## 1. What and Why of DevOps?

### What is DevOps?
DevOps is a set of practices that combines software development (Dev) and IT operations (Ops). It aims to shorten the systems development life cycle and provide continuous delivery with high software quality. It is a culture shift that breaks down "silos" between teams.

### Why DevOps?
- **Speed**: Innovate for customers faster and grow more efficient.
- **Rapid Delivery**: Increase the frequency and pace of releases to improve the product.
- **Reliability**: Ensure the quality of updates through automated testing.
- **Security**: Move quickly while retaining control and preserving compliance.

## 3. Step-by-Step Implementation

### Step 3.1: Source Control Management (SCM)
**Brief**: SCM is used to track every change made to the code. GitHub provides a cloud-based home for our code.

**Setup & Configuration**:
1. Create a repository on GitHub named `calculator`.
2. Map the local folder to GitHub using `git remote add origin`.
3. Configure `.gitignore` to avoid pushing temporary files.

**Commands**:
```bash
git init
git add .
git commit -m "First commit: Scientific Calculator"
git remote add origin https://github.com/AbhashTi/calculator.git
git push -u origin main
```

> Please refer to `screenshots/github_repo.png` to see the repository setup and the initial push.

---

### Step 3.2: Automated Build (Maven)
**Brief**: Maven takes the raw Java code and transforms it into a "JAR" file that can actually be executed.

**Configuration**:
The `pom.xml` file defines all dependencies (like Spring Boot) needed for the calculator.

**Commands**:
```bash
./mvnw clean package
```

---

### Step 3.3: Continuous Integration (Jenkins)
**Brief**: Jenkins is the "engine" that runs every time we push code. It automates testing and building.

**Setup & Configuration**:
1. **Pipeline Creation**: Click "New Item" -> "Pipeline" -> Name it `Calculator`.
2. **Credential Management**: Go to "Manage Jenkins" -> "Credentials". Add Docker Hub credentials as `dockerhub-creds`.
3. **Webhook**: Connect GitHub to Jenkins so it triggers on every push.

**Jenkinsfile Script**:
```groovy
pipeline {
    agent any
    stages {
        stage('Stage 1: Git Clone') { steps { checkout scm } }
        stage('Stage 2: Maven Build') { steps { sh 'mvn clean package' } }
         stage('Stage 4: Build Docker Image') { 
            steps { sh 'docker build -t abhashti/calculator:latest .' } 
        }
        stage('Stage 5: Push Docker Image') {
            steps {
                withCredentials([usernamePassword(credentialsId: 'dockerhub-creds', ...)]) {
                    sh 'docker push abhashti/calculator:latest'
                }
            }
        }
    }
}
```

> Refer to `screenshots/jenkins_pipeline.png` for the visual dashboard of the successful build.
> Refer to `screenshots/jenkins_credentials.png` for the credential configuration details.

### Step 3.4: Containerization (Docker)
**Brief**: Docker wraps the application in a "container" so it runs precisely the same way on any computer.

**Setup**: Create a file named `Dockerfile` in the root directory.

**Dockerfile Script**:
```dockerfile
FROM eclipse-temurin:17-jre
WORKDIR /app
COPY target/*.jar app.jar
EXPOSE 8080
ENTRYPOINT ["java", "-jar", "app.jar"]
```

**Commands**:
```bash
docker build -t abhashti/calculator:latest .
docker push abhashti/calculator:latest
```
> [!NOTE]
> Refer to `screenshots/docker_hub.png` to see the image successfully hosted in the cloud.

---

### Step 3.5: Deployment (Ansible)
**Brief**: Ansible is a configuration tool that tells our remote server to pull the image and start the calculator.

**Setup & Configuration**:
1. **inventory.ini**: Defines the target IP (localhost/remote).
2. **deploy.yml**: The "playbook" containing deployment logic.

**Ansible Playbook (deploy.yml)**:
```yaml
- hosts: localhost
  tasks:
    - name: Pull latest image
      shell: docker pull abhashti/calculator:latest
    - name: Clear port 8081
      shell: |
        docker stop Calculator || true
        docker rm Calculator || true
    - name: Run Scientific Calculator
      shell: docker run -d --name Calculator -p 8081:8080 abhashti/calculator:latest
```
