## 1. What and Why of DevOps?

### What is DevOps?
DevOps is a set of practices that combines software development (Dev) and IT operations (Ops). It aims to shorten the systems development life cycle and provide continuous delivery with high software quality. It is a culture shift that breaks down "silos" between teams, encouraging shared responsibility and transparency.

### Why DevOps?
- **Speed**: Move at high velocity so you can innovate for customers faster, adapt to changing markets better, and grow more efficient at driving business results.
- **Rapid Delivery**: Increase the frequency and pace of releases so you can innovate and improve your product faster.
- **Reliability**: Ensure the quality of application updates and infrastructure changes so you can reliably deliver at a more rapid pace while maintaining a positive experience for end users.
- **Scale**: Operate and manage your infrastructure and development processes at scale. Automation and consistency help you manage complex or changing systems efficiently and with reduced risk.

## 3. Pipeline Stages Explanation

The CI/CD pipeline is designed to automate the lifecycle of the Scientific Calculator. Each stage ensures a specific quality gate is passed before moving to the next.

### Stage 1: Source Control Management (SCM)
This is the entry point of the pipeline. Whenever a developer pushes code to the GitHub repository, a Webhook signals Jenkins to start the build. Jenkins clones the latest version of the code to its workspace. This ensures that the automation always works on the most recent updates.

### Stage 2: Maven Build
In this stage, the Java code is compiled. Maven checks for syntax errors and ensures all external libraries (dependencies) are correctly downloaded. If the compilation fails, the pipeline stops here, preventing broken code from reaching production.

### Stage 3: Unit Testing
Automated tests are executed to verify the mathematical accuracy of the calculator (e.g., Square Root, Factorial). Jenkins parses these test results; if any test fails, the build is marked as "Unstable" or "Failed," ensuring that logical bugs are caught immediately.

### Stage 4: Package
Once the code is verified, Maven packages the application into an executable JAR (Java Archive) file. This file contains the entire application and is the "artifact" that will eventually run in the container.

### Stage 5: Docker Build & Tag
Jenkins uses the `Dockerfile` to create a Docker image. This image includes the OS, the Java runtime, and the application JAR. It is then tagged (e.g., `latest`) to signify it is the newest version ready for release.

### Stage 6: Docker Push (Distribution)
The built image is uploaded to **Docker Hub**. This acts as a central registry. By pushing the image to the cloud, any server (production, testing, or a teammate's computer) can download and run the exact same version of the calculator.

### Stage 7: Ansible Deployment
The final stage uses Ansible to communicate with the target deployment server. It performs three critical sub-steps:
1. **Cleanup**: It identifies and stops any previous versions of the calculator running on port 8081 to avoid "Port already in use" errors.
2. **Pull**: It downloads the fresh image from Docker Hub.
3. **Run**: It starts a new container, making the calculator's web interface accessible to users.

## 4. Final Result & Operation
The scientific calculator is now accessible via a web browser. A user can simply navigate to the provided IP address, enter a number, and click the desired operation.

### Option A: Remote Server (Production)

- **Commands**:
  ```bash
  docker pull abhashti/calculator:latest
  docker run -d --name scicalc -p 8080:8080 abhashti/calculator:latest
  ```
- **URL**: [http://localhost:8080/](http://localhost:8080/)

### Option C: Local Host (Source Code)
For users who want to run the project directly from the source code:
1. **Clone the Project**:
   ```bash
   git clone https://github.com/AbhashTi/calculator.git
   cd calculator
   ```
2. **Run with Maven**:
   ```bash
   ./mvnw spring-boot:run
   ```
3. **Access**: Open [http://localhost:8080/](http://localhost:8080/) in your browser.
